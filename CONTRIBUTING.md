# Contributing to Tunecamp

Thank you for your interest in contributing! We welcome help in various forms: bug reports, feature requests, documentation improvements, and pull requests.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub and provide as much detail as possible, including your environment and steps to reproduce.

### Feature Requests
Have an idea for a new feature? Open an issue to discuss it before starting work.

### Pull Requests
1.  **Fork the repo**: Create a fork on GitHub.
2.  **Clone it locally**: `git clone https://github.com/your-username/tunecamp.git`.
3.  **Install dependencies**: `npm install`.
4.  **Create a branch**: `git checkout -b feature/my-new-feature`.
5.  **Develop & Test**: Ensure your changes follow the project's style and all tests pass.
6.  **Submit a PR**: Push your branch to GitHub and open a pull request.

### Coding Style
- We use **TypeScript** for both backend and frontend.
- Maintain a clean, modular structure.
- Document new features or configuration variables.
